# Middleton Finance Suite — Supply Chain Edition

An extension of the Middleton Finance Suite covering Supply Chain Finance analytics — demand planning, inventory health, COGS structure, and gross margin analysis. Built to demonstrate cross-functional finance capability alongside the existing Sales Finance suite.

**Live demo → [bradymiddleton.github.io/middleton-finance-suite](https://bradymiddleton.github.io/middleton-finance-suite)**

---

## What This Is

This module simulates the analytical workflow of a Supply Chain Finance analyst supporting a global operations organization. It covers three core workstreams:

| Tool | What It Does |
|------|-------------|
| **Demand Planning & Forecast Accuracy** | MAPE and bias analysis by SKU and category, forecast vs. actual variance, worst-performing SKU identification |
| **Inventory Health & Holding Cost** | DIO vs. target by category and warehouse, holding cost breakdown (capital, storage, insurance, obsolescence), slow mover and write-off risk |
| **COGS & Gross Margin** | Full margin waterfall (list → discount → net revenue → COGS → GP), component breakdown, standard vs. actual cost variance by category and supplier |
| **SC AI Finance Assistant** | Chat interface querying all three SC databases, generating analyst commentary grounded in real FY2024 values |

## The Data

All data is **fully synthetic** — generated programmatically so the project runs locally without any API keys or Kaggle authentication. Schemas and business logic are modeled after three real public datasets:

| Database | Rows | Tables | Modeled After |
|----------|------|--------|---------------|
| `demand_planning.db` | 648 | sku_forecast, actuals, forecast_accuracy | [Product Demand Forecasting](https://www.kaggle.com/datasets/felixzhao/productdemandforecasting) · [Demand Forecast for Inventory](https://www.kaggle.com/datasets/oscarm524/demand-forecast-for-optimized-inventory-planning) |
| `inventory.db` | 2,808 | inventory_positions, holding_costs, slow_movers | [Supply Chain Dataset](https://www.kaggle.com/datasets/amirmotefaker/supply-chain-dataset) · [Retail Store Inventory](https://www.kaggle.com/datasets/anirudhchauhan/retail-store-inventory-forecasting-dataset) |
| `cogs_margin.db` | 648 | cost_breakdown, standard_vs_actual, margin_waterfall | [Logistics and Supply Chain](https://www.kaggle.com/datasets/datasetengineer/logistics-and-supply-chain-dataset) |

The data covers:
- **18 SKUs** across 5 product categories (GPUs, CPUs, Embedded, Semi-Custom, Adaptive Computing)
- **6 warehouses** (Dallas TX, Portland OR, Columbus OH, Atlanta GA, Amsterdam NL, Singapore SG)
- **6 suppliers** (TSMC, Samsung Foundry, UMC, GlobalFoundries, ASE Group, Amkor Technology)
- **FY2024** full year with realistic seasonal patterns, systematic forecast bias, and cost variance

## SQL Analysis

`06_sc_queries.sql` contains 18 standalone queries organized into 4 sections. Every query is labeled, commented, and explains what finance question it answers.

```
SECTION 5 — Demand Planning & Forecast Accuracy    [demand_planning.db]
  5A  Forecast accuracy (MAPE) by product category
  5B  Monthly forecast vs. actual by category
  5C  Worst-performing SKUs by MAPE — top 10
  5D  Forecast bias analysis — systematic over/under by category
  5E  Quarterly revenue variance — forecast vs. actual

SECTION 6 — Inventory Health & Holding Cost        [inventory.db]
  6A  Days Inventory Outstanding (DIO) vs. target by category
  6B  Total holding cost by category and component
  6C  Monthly holding cost trend
  6D  Slow movers and excess inventory by category
  6E  Inventory position by warehouse

SECTION 7 — COGS & Gross Margin Analysis           [cogs_margin.db]
  7A  Gross margin waterfall by category
  7B  COGS component breakdown by category
  7C  Standard vs. actual cost variance by category
  7D  Monthly COGS and margin trend
  7E  Supplier cost variance — which suppliers drive overruns

SECTION 8 — Cross-Database Snapshots               [all three DBs]
  8A  Supply chain health summary
  8B  Inventory health snapshot
  8C  COGS health snapshot
```

To run all queries:

```bash
python 07_run_sc_queries.py
```

## Excel Financial Model

`sc_sales_finance_model.xlsx` extends the Sales Finance workbook with supply chain sheets:

| Sheet | What It Contains |
|-------|-----------------|
| **SC_Assumptions** | Holding cost rates (cost of capital 8%, storage 2%, insurance 0.5%, obsolescence 1.5%), DIO targets by category, standard cost inputs — all blue-text inputs |
| **SC_Data_Forecast** | Raw sku_forecast + actuals CSV data loaded as Excel tables |
| **SC_Data_Inventory** | Raw inventory_positions + holding_costs data |
| **SC_Data_COGS** | Raw margin_waterfall + standard_vs_actual data |
| **Demand Planning** | MAPE and bias by category (AVERAGEIF from SC_Data_Forecast), quarterly variance, worst SKUs |
| **Inventory & Holding** | DIO vs. target (AVERAGEIF), holding cost by component (SUMIF), slow mover risk |
| **COGS & Margin** | Margin waterfall by category (SUMIF), component mix %, standard vs. actual variance (XLOOKUP) |

**Color coding standard:**

| Color | Meaning |
|-------|---------|
| Blue text | Hardcoded input — editable for scenario analysis |
| Black text | Formula — do not edit directly |
| Green text | Cross-sheet reference link |
| Yellow background | Key assumption requiring periodic review |

## Dashboards

All four dashboards are standalone HTML files — no server, no build step. They share the same nav bar as the Sales Finance suite (Dashboards 1–4), with a divider separating the two suites.

**Dashboard 5 — Demand Planning & Forecast Accuracy**
- KPI cards: blended MAPE, forecast bias, total actual revenue, units shipped
- Horizontal bar chart: MAPE by category (color-coded green/amber/red)
- Line chart: monthly forecast vs. actual units — FY2024
- Tables: accuracy by category, quarterly revenue variance, worst 10 SKUs by MAPE
- AI panel: grounded commentary on bias patterns and improvement priorities

**Dashboard 6 — Inventory Health & Holding Cost**
- KPI cards: total inventory value, avg DIO, monthly holding cost, write-off risk
- Bar/line chart: DIO vs. target by category
- Stacked bar chart: monthly holding cost by component (capital, storage, insurance, obsolescence)
- Tables: DIO variance by category, holding cost breakdown, slow movers, warehouse rankings

**Dashboard 7 — COGS & Gross Margin**
- KPI cards: total net revenue, total COGS, gross profit, blended GM%
- Bar/line combo: monthly gross margin % trend with net revenue overlay
- Stacked bar: COGS component mix by category (materials, labor, overhead, logistics, yield loss)
- Tables: margin waterfall, component breakdown %, standard vs. actual variance, supplier cost variance

**Dashboard 8 — SC AI Finance Assistant**
- Dark full-screen chat interface connected to all 3 SC databases
- 6 supply chain-specific prompts (MAPE, DIO, holding cost, COGS, variance, supplier)
- Mock AI responses grounded in real FY2024 values from all three databases
- Pre-loaded API key field (demo mode — no live API calls)

## How to Run Locally

**Prerequisites:**

```bash
pip install pandas numpy faker openpyxl
```

**Step 1 — Generate SC data and build databases:**

```bash
python 05_generate_sc_data.py
```

Builds all three SQLite databases in `sc_databases/` and exports CSVs to `sc_exports/`.

**Step 2 — Run SQL analysis:**

```bash
python 07_run_sc_queries.py
```

Executes all 18 queries and prints formatted results to console.

**Step 3 — Open dashboards:**

Open any of the `sc_0*.html` files in a browser. All four dashboards are linked via the shared nav bar.

> Note: Uses a fixed random seed (`numpy.random.seed(99)`) — every run produces the same data.

## Key Metrics Explained

| Metric | Definition | Target |
|--------|-----------|--------|
| **MAPE** | Mean Absolute Percentage Error — average forecast error as % of actuals | < 10% (good), 10–20% (acceptable), > 20% (poor) |
| **Forecast Bias** | Systematic over- or under-forecasting direction | 0% (balanced) |
| **DIO** | Days Inventory Outstanding — how many days of supply on hand | Category-specific (28–60 days) |
| **Inventory Turns** | Annualized COGS ÷ average inventory value | Higher = more efficient |
| **Holding Cost Rate** | Annual cost of carrying inventory as % of value | 20–28% industry benchmark |
| **Standard vs. Actual Variance** | Actual unit cost minus standard (budgeted) cost | 0 (favorable = negative) |
| **Yield Loss** | Cost of defective or scrapped production units | Category-specific |
